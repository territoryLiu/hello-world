# 旅游规划页面写作与设计手册

适用场景：以后继续做别的旅游规划、旅游攻略页、单文件 HTML 旅行专题页时，直接参考这份手册即可。

## 目标

做出来的页面要满足 4 件事：

- 一眼能看懂重点，不要让读者先适应页面风格。
- 内容足够实用，能直接照着安排行程。
- 语气温暖、周到，像一个会提前想到坑点的旅行管家。
- 设计服务内容，不要让设计喧宾夺主。

## 核心原则

- 内容优先，不写“作者在解释自己怎么写”的句子。
- 页面里的每一句话都应该对出行有帮助，或者明确改善阅读体验。
- 先讲天气和穿衣，再讲路线和交通，再讲扩展玩法。
- 可以温暖，但不要啰嗦。
- 可以有一点口头感，但不要把页面写成聊天记录。
- 允许少量口头禅点缀气氛，比如“这一定很爽”“太酷啦”，但只能出现在适合的场景推荐里，不能满篇乱飘。

## 明确不要写的内容

下面这类句子默认不要出现在正文里：

- “我先替你想到前面”
- “这里不是单篇笔记摘抄”
- “这版不再像冷冰冰的清单”
- “我把这份攻略整理成……”
- “下面是我综合搜索后的结果”
- 任何在解释写作过程、搜索过程、排版过程的话

页面正文只交付结果，不交代写作幕后。

## 推荐口吻

推荐的语气是：

- 温暖
- 克制
- 周到
- 有轻微服务感
- 像提醒你别忘带外套、别把早市排太晚、别把同一天塞太满的人

推荐的表达方式：

- “更稳的安排是……”
- “这一天建议留一点机动时间。”
- “如果只选一版执行，可以直接走这条线。”
- “这里很适合放在傍晚。”
- “上山这天别和天气赌运气。”

不推荐的表达方式：

- 过于官方
- 过于命令
- 过于自我感动
- 过于解释型

## 页面结构

以后做旅游规划页，默认用下面这套结构：

### 1. 页面标题

最上方只放一个清楚的标题即可。

推荐格式：

`假期旅游攻略：[城市 A] · [城市 B] · [景区 C]`

不要在标题区堆太多解释文字。头图不是必须项。如果做头图，也要极简，不能把一大堆说明塞在首屏。

### 2. 穿衣指南

这是默认第一部分。

要写：

- 目的地在出行日期附近的温度判断
- 城市穿什么
- 山区 / 景区 / 早晚穿什么
- 一份简单打包清单
- 一句最重要的穿衣提醒

如果路线里同时有城市和山地，必须分开写，不要笼统地给一套穿搭建议。

### 3. 最推荐路线和交通方式

这是默认第二部分。

要给出一版最推荐执行方案，并且配一张单页执行表。

执行表字段固定为：

- 日期
- 时间段
- 交通
- 景点
- 吃什么
- 住哪里

如果读者只想照着走，这一张表就要够用。

### 4. 最全面的旅游攻略

这是默认第三部分，也是最完整的信息库。

建议包含：

- 多种路线方案
- 火车 / 飞机 / 客运 / 拼车 / 包车等交通方式
- 多地点停留建议
- 预算区间
- 多地点景点备选
- 多地点美食备选
- 店名清单
- 高频避坑提醒

这一部分的原则是：让读者可以自行挑选和组合。

### 5. 注意事项

这是默认第四部分。

集中写：

- 哪些点容易翻车
- 哪些安排容易太累
- 哪些票要先订
- 哪些天气风险需要预留机动时间
- 哪些区域更适合住

注意事项要写得像提醒，不要写得像训话。

### 6. 信息来源

这是默认第五部分。

所有搜索、核实、调研，都收束到最后这一块。

正文里不需要不断解释“这是从哪里搜来的”，正文只讲结论。

## 调研要求

以后做任何旅游攻略页，默认按这套调研顺序来：

### 官方和一手信息

优先查：

- 景区官方
- 文旅局 / 本地文旅
- 官方交通平台
- 官方天气与历史天气
- 官方景区预约规则

这些内容主要用于：

- 门票规则
- 开放时间
- 预约方式
- 官方交通 / 景区交通
- 节假日政策

### 小红书调研

小红书必须看，而且不是只看标题。

要看两部分：

- 正文
- 评论区

正文主要拿来提炼：

- 高频路线
- 热门景点
- 店名
- 景区顺序
- 好拍照 / 好吃 / 好逛的具体点位

评论区主要拿来提炼：

- 排队情况
- 实际是否累
- 哪些点是否值得
- 哪些安排容易踩坑
- 实际交通体验
- 实际天气体感

不要只看高赞标题党，要看评论里反复出现的真实反馈。

### 全网搜索

全网搜索用来补齐：

- 公开媒体和地方媒体报道
- 本地生活站点
- 交通参考页
- 餐厅公开页面
- 历史天气页

如果小红书里出现了具体店名，尽量再找公开页面去对上，避免只凭单平台笔记硬写。

## 小红书内容怎么用

把小红书内容转成“可执行结论”，不要原样搬运。

正确用法：

- “评论区普遍反馈东方水上市场要尽早去，晚了更多是在排队。”
- “正文和评论区都反复提到，北坡前一晚住二道白河会更稳。”
- “多篇笔记都把网红墙和延边大学放在同一晚安排。”

错误用法：

- “有一篇笔记说……”
- “某博主推荐……”
- “我看到小红书上很多人说……”

## 美食块怎么写

美食部分默认并入“最全面的旅游攻略”，不要孤零零拆成特别松散的一块。

推荐写法：

- 城市 A 吃什么
- 城市 B 吃什么
- 景区落脚地吃什么
- 按时间段怎么吃
- 店名备选库

如果信息够多，直接做城市维度的店名库。

推荐数量：

- 主要城市：5 到 8 家
- 次级落脚地：3 到 5 家

每家店建议写三件事：

- 店名
- 适合吃什么
- 更适合安排在哪一顿

## 设计原则

设计方向会变，但有几条长期有效：

- 首屏不要写成说明书。
- 不要让头图过空，也不要塞爆。
- 如果标题已经够清楚，头图可以只留一个标题。
- 设计必须帮读者更快找到重点。
- 颜色和样式要统一，不要一半像手账，一半像 App。

## 目前验证过的用户偏好

基于这次吉林攻略的来回迭代，可以总结出下面这些偏好：

- 比起解释型头图，更喜欢内容直接开始。
- 比起“我在帮你写攻略”的说法，更喜欢结果导向的页面。
- 比起花哨大首屏，更接受简单标题 + 后面内容完整。
- 喜欢温暖感，但不喜欢“过度自我说明”。
- 喜欢有服务意识的提醒，但不喜欢写作者存在感太强。
- 如果设计不好看，宁可回退，也不要强行保留新风格。

## 口头禅使用规则

“这一定很爽”“太酷啦”这类口头禅可以用，但只能小剂量使用。

适合放在：

- 某个轻松玩法的推荐句
- 某个景点 / 夜游 / 美食的情绪点缀
- 某个小节标题旁的短句

不适合放在：

- 门票规则
- 天气判断
- 交通信息
- 风险提醒
- 预算说明

原则是：点一下就够，不能每屏都喊。

## 每次生成前的自检清单

开始写之前，先检查这几项：

- 标题是否已经足够清楚
- 第一部分是不是穿衣指南
- 有没有一版最推荐路线
- 有没有单页执行表
- 有没有最全面的攻略库
- 有没有注意事项
- 信息来源是不是全部放到了最后
- 正文里有没有“我怎么写”“我怎么搜”的句子
- 小红书是否同时看了正文和评论区
- 店名是否尽量有公开页面可核

## 可复制的旅游规划提示词模板

下面这段可以直接复制，替换目的地后使用。

```md
请帮我做一份 [假期名称] [目的地 A] · [目的地 B] · [目的地 C] 旅游攻略，输出为单文件 HTML 页面。

要求：

1. 页面最上方只放一个清楚标题，不要写大段头图说明。
2. 正文语气温暖一点，像一个处处替人想到前面的旅行管家，但不要写成作者在解释自己怎么写。
3. 删除所有“写作说明式”的句子，不要出现“我先替你想到前面”“这里不是单篇笔记摘抄”“我整理了这份攻略”这类表达。
4. 第一部分必须是穿衣指南，要结合出行日期和目的地天气来写，区分城市和景区 / 山区穿衣。
5. 第二部分是最推荐的旅游路线和交通方式，要给出一版最推荐方案，并做一张单页执行表，字段固定为：日期 / 时间段 / 交通 / 景点 / 吃什么 / 住哪里。
6. 第三部分是最全面的旅游攻略，要包含多方案、多交通方式、多地点、多美食、多店名备选，让人可以自行选择。
7. 第四部分写注意事项，集中写容易翻车、需要提前订、容易太累、需要留机动时间的点。
8. 第五部分统一写信息来源，正文不要一直解释来源。
9. 必须继续搜索小红书和全网信息，小红书正文和评论区都要看，提炼高频结论，不要只看标题。
10. 页面里可以少量加入“这一定很爽”“太酷啦”这种口头禅，但只能点到为止，不能乱用。
11. 设计上以内容优先为主，保证好看和好读，但不要让设计喧宾夺主。

请按这个结构输出：

- 标题
- 穿衣指南
- 最推荐的旅游路线和交通方式
- 最全面的旅游攻略
- 注意事项
- 信息来源
```

## 最后一句

以后做旅游规划页，先把页面当成一个“能直接拿去用”的工具，而不是一篇“我写得很用心”的文章。工具做对了，读者自然会觉得贴心。
