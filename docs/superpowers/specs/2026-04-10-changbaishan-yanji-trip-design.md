# 吉林延吉长白山 Trip 页面改版设计

日期：2026-04-10

## 目标

将现有的 [吉林延吉长白山trip.html](/d:/vscode/hello-world/吉林延吉长白山trip.html) 从“信息全但视觉发散”的单文件攻略页，改成更统一、更高级、也更容易读的旅行专题页。

本次设计分为两条交付线，同时保留：

1. 原文件覆盖版：只优化视觉和版式，不改变正文内容顺序和原有锚点逻辑。
2. 重组副本版：另存新文件，在同一视觉体系下重组信息层级，改成“先结论，后展开”的专题阅读流。

## 成功标准

- 页面第一眼风格接近参考图的气质：
  - 奶油暖底
  - 局部深色高对比面板
  - 杏黄高亮点缀
  - 圆角、大留白、柔和投影
- 版式阅读感接近旅行杂志，而不是密集小卡片拼盘。
- 保持单文件离线 HTML，不引入外链脚本、外链字体或构建步骤。
- 手机和桌面端都可正常阅读。
- 重组副本版相较原版，开头 1 屏到 2 屏内就能看完“怎么走、怎么穿、怎么订、住哪儿”。

## 用户已确认的设计方向

用户确认采用：

- `A` 的配色：奶油暖底、炭黑对比、杏黄点亮。
- `B` 的排版：更像旅行杂志专题，而不是仪表盘式密集卡片堆叠。

因此最终设计不是直接复刻参考图，而是提取其配色和对比关系，再落到适合旅行攻略阅读的专题排版中。

## 共用视觉系统

两版页面共用同一套视觉语言。

### 1. 色彩

- 背景主色：暖奶油、浅米、纸感米白。
- 主文字：偏炭黑或深褐灰，保证阅读对比。
- 高亮色：杏黄或暖金，用于按钮、重点数字、标签、节点。
- 辅助色：鼠尾草绿、雾感橄榄灰，承担山系气质。
- 深色面板：用于关键摘要、时间轴或路线总览，制造节奏。

### 2. 版式

- 首屏改成“标题 + 摘要 + 关键信息条”的杂志式开场。
- 卡片数量减少，模块变大，避免每段都像一个小便签。
- 标题层级更鲜明，正文列宽更克制，提升长文阅读舒适度。
- 重要表格和路线内容保留，但包装成更整齐的专题模块。

### 3. 组件语义

- `hero`：视觉主引导区，承担题眼、适用时间、路线基调。
- `summary rail`：首屏摘要条，快速给出目的地、气温层次、推荐玩法、订票顺序。
- `feature panel`：大模块容器，用于承载推荐路线、穿衣结论、注意事项等重点内容。
- `editorial section`：普通专题段落，承载攻略正文和表格。
- `dark panel`：小面积深色区块，用于关键结论、路线摘要或执行提醒。

### 4. 交互与动效

- 保持轻量，优先 CSS。
- 增加轻微 hover、卡片浮起、锚点导航反馈。
- 不做重 JS 动画，不做依赖脚本的滚动特效。

## 版本一：原文件覆盖版

对应文件：

- [吉林延吉长白山trip.html](/d:/vscode/hello-world/吉林延吉长白山trip.html)

### 目标

在不改变正文顺序、不重命名主要内容段、不打乱现有阅读路径的前提下，把现有页面的视觉质量整体抬高。

### 保留项

- 现有主标题和整体内容框架。
- 现有锚点分区：
  - 穿衣指南
  - 最推荐路线
  - 最全面攻略
  - 注意事项
  - 信息来源
- 现有正文内容顺序、表格核心信息和链接。

### 调整项

- 重写 `:root` 视觉令牌，统一色板、阴影、边框和圆角。
- 重做首屏头部，让标题区更像专题封面，而不是简单的单行标题。
- 将导航做成更克制的浮层式胶囊导航，减少彩色按钮碎片感。
- 把各 section 从“均质卡片堆叠”改成“标题区 + 重点结论 + 正文模块”结构。
- 降低大面积虚线边框和多色渐变的存在感，避免页面看起来像手账拼贴。
- 优化表格、列表、说明文字的层级和留白。

### 不做的事

- 不删减信息。
- 不改正文顺序。
- 不新增联网依赖。
- 不把页面改成需要构建的项目。

## 版本二：重组副本版

对应文件：

- 新增一个并列 HTML 副本文件，基于现有内容重组层级。

建议命名：

- [吉林延吉长白山trip-专题版.html](/d:/vscode/hello-world/吉林延吉长白山trip-专题版.html)

如果实现时发现文件名需要更稳妥的 ASCII 方案，可改成语义相近的中文或拼音文件名，但需保持与原文件并存，不覆盖原文件。

### 目标

把现有页面改造成更像“给朋友直接发的专题攻略”，让用户先拿到执行答案，再决定要不要继续往下读细节。

### 新阅读流

1. 首屏封面
2. 先看这版结论
3. 最推荐路线时间轴
4. 穿衣和打包一句话版
5. 订票顺序和住宿落点
6. 展开版完整攻略
7. 注意事项
8. 信息来源

### 信息重组原则

- 所有原始信息都尽量保留，但允许去重。
- 把重复表达合并，避免同一结论在多个区块来回出现。
- 先给“最适合第一次去的人”的推荐主方案，再给 Plan A/B/C 扩展。
- 来源统一放到页面末尾，不在前半段打断阅读。

### 首屏必须回答的四个问题

- 怎么走最稳
- 五一到底怎么穿
- 延吉和长白山分别住哪里
- 哪些票和房先订

### 可以主动压缩的内容

- 语义重复的铺垫性句子
- 过多相似结构的卡片容器
- 视觉上没有必要拆开的平行模块

## 文件与实现边界

### 交付文件

- 覆盖更新：[吉林延吉长白山trip.html](/d:/vscode/hello-world/吉林延吉长白山trip.html)
- 新增副本：[吉林延吉长白山trip-专题版.html](/d:/vscode/hello-world/吉林延吉长白山trip-专题版.html)

### 技术边界

- 仅使用原生 HTML + CSS。
- 允许少量原生 JS，但仅在确有必要时使用。
- 默认不引入外链资源。
- 保持单文件可直接双击打开。

## 验证方式

- 视觉检查：桌面端和窄屏端都不出现明显错位。
- 结构检查：原文件覆盖版保留原锚点可跳转。
- 内容检查：重组副本版首屏至前两屏能读到核心结论。
- 兼容性检查：在本地浏览器直接打开可正常渲染。

## 风险与注意点

- 原文件当前已经有未提交修改，实现时不能误覆盖掉不相关变更。
- 重组副本版若过度删减，容易损失“保姆级攻略”的信息密度，因此只能做去重和排序优化，不能偷工减料。
- 当前仓库尚未忽略 `.superpowers/` 目录，后续提交时要避免把 brainstorming 临时文件带进版本库。

## 实现建议

- 先完成版本一，确保现有页面马上变好看。
- 再基于版本一的视觉系统复制出版本二，减少两套样式分叉。
- 两版尽量复用同一套变量和组件思路，但分别落在各自单文件内，避免相互依赖。
