# HTML Visual Rules

## Page Rhythm

- Start with a cover or lead area that establishes trip mood, route promise, and destination character
- Introduce each major section with a heading block or short editorial lead
- Let the main content body carry the factual depth without flattening into identical modules
- Treat sources as restrained trust signals rather than dominant UI blocks

## Section-Specific Rules

- `recommended_route`: use a hero summary, route promise, and narrative timeline
- `route_options`: use a comparison layout that emphasizes trade-offs and decision criteria
- `clothing_guide`: present layered wear guidance, temperature context, and scene-based advice rather than e-commerce cards
- `attractions`: organize by experience sequence, scene quality, and pitfalls; avoid waterfall-card galleries
- `transport_details`: make operational facts highly scannable with strong hierarchy for time, transfer, and price
- `food_by_city`: allow a more lifestyle-oriented tone while staying inside the same visual system
- `tips`: use side-notes, warning blocks, and field-note patterns
- `sources`: keep them compact, trustworthy, and easy to inspect

## Responsive Rule

- Desktop and mobile must share the same facts
- Mobile may split long reading flows into smaller segments without changing section order
- Responsive changes should improve pacing, not replace the editorial baseline with app-like cards
